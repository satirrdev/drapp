# Dr. App — Project Knowledge

Neo-brutalist web app for browsing open-source Android apps from the [F-Droid](https://f-droid.org) repository. Built with Vite + React 19, using client-side hash routing. All app data is pre-generated into static JSON files at build time and cached in `localStorage` at runtime.

## Quickstart
- **Install:** `npm install`
- **Dev:** `npm run dev` (runs `prepare-data.js` then starts Vite)
- **Build:** `npm run build` (runs `prepare-data.js` then `vite build`)
- **Preview:** `npm run preview`
- **Data only:** `npm run prepare-data`
- **Test/Lint:** none configured.

## Architecture
- **`scripts/prepare-data.js`** — Build-time data prep. Downloads the F-Droid `index-v1.json` (falls back to `public/apps-fallback.json` if the download fails), then writes to `public/`:
  - `page_N.json` — paginated card data, 20 apps/page (compact keys: `id`, `n`=name, `s`=summary, `i`=icon, `v`=version, `c`=categories).
  - `data/<pkg>.json` — per-app detail (`d`=description, `src`, `lic`, `web`, `up`, `vc`, `a`=author, etc.). Package names have `/` sanitized to `_`.
  - `search_index.json` — lightweight name+category index for search.
- **`src/App.jsx`** — Root. `HashRouter` with routes: `/` → `/page/1`, `/page/:num`, `/app/:appId`. Manages light/dark theme (persisted in `localStorage` key `dr-app-theme`, applied via `data-theme` attr on `<html>`).
- **`src/pages/`** — `Page.jsx` (paginated list), `AppDetail.jsx` (single app view).
- **`src/components/`** — `Header`, `ActionBar`, `AppCard`, `Pagination`, `SearchBar`.
- **`src/services/fdroidApi.js`** — Client-side data fetching (page files, detail files, search index).
- Icons lazy-loaded (native `loading="lazy"` / IntersectionObserver) for perf on slow devices.

## Conventions & Gotchas
- **Hash routing:** Always use React Router `<Link>` for internal navigation — never plain `<a>` — to stay inside `HashRouter`.
- **Deploy targets:** `vite.config.js` switches on the `VERCEL` env var:
  - Vercel: `base: '/'`, `outDir: 'dist'` (build forced via `vercel.json` → `npm run build`).
  - GitHub Pages: `base: '/drapp/'`, `outDir: 'docs'`.
  - When fetching static assets, prefix paths with `import.meta.env.BASE_URL` so both subpath and root deploys work.
- **Compact JSON keys:** Page/detail JSON use single-letter keys to shrink payloads; keep the mapping in `prepare-data.js` and `fdroidApi.js` in sync.
- **F-Droid icon URLs** are built from `https://f-droid.org/repo/...`; localized icon preferred over the direct `/icons/` path.
- **Neo-brutalist UI:** thick black borders, hard (non-blurred) shadows, monospace fonts, no rounded corners — keep new UI consistent with this.
- The `docs/` folder is committed build output for GitHub Pages; regenerated by `npm run build`.
